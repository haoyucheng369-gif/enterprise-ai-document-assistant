# Common

Common modules contain cross-cutting application infrastructure.

- `Audit`: in-process audit events and activity records.
- `Guardrails`: request and response safety checks.
- `Harness`: repeatable validation checks for prompts and tools.
- `Options`: strongly typed configuration.
- `Services`: small shared services that do not belong to one feature.
- `Storage`: persistence adapters such as MongoDB, SQL Server, or vector store clients.
- `Swagger`: OpenAPI examples and filters.
